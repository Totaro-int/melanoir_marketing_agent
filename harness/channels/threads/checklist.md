# Threads 발행 전 체크리스트

휴먼 승인 게이트(`/sns-preview <slug>` → approve)에서 사용. `copywriter` 산출물에 대해 자동 + 수동 점검.

## 자동 점검 (Phase 3 게이트가 차단)

- [ ] 본문 길이 ≤ 500자 (한국어)
- [ ] 첫 줄 ≤ 80자 (피드 노출 컷)
- [ ] 해시태그 개수 ≤ 3
- [ ] `banned.words` 미포함
- [ ] `banned.claims` 미포함 ("최고의", "1위", "유일한" 등)
- [ ] `banned.topics` 미포함
- [ ] 외부 링크가 본문 첫 줄에 없음
- [ ] `hashtags.always`가 모두 포함됨
- [ ] (광고 분류 시) `legal.adHashtag` 포함

## 수동 점검 (사람이 확인)

- [ ] 첫 줄이 스크롤을 멈출 만큼 흥미로운가?
- [ ] 회사 톤(`tone.voiceNotes`)과 일치하는가?
- [ ] 광고 카피처럼 들리지 않는가?
- [ ] 사실 확인 — 숫자·인용·날짜
- [ ] 미디어가 본문과 어울리는가? (카드뉴스의 텍스트 가독성 모바일 기준)
- [ ] 답글이 달릴 만한 여지(질문·관점)가 있는가?

## 시리즈/스레드인 경우 추가

- [ ] 1편이 단독으로도 가치가 있는가? (스레드 안 펴는 사용자 다수)
- [ ] 2편 이후가 1편 클릭 유인이 되는가?
- [ ] 마지막 편에 명확한 마무리·CTA 1개

## 실패 시 행동

- 자동 점검 실패: 사용자에게 어느 룰이 깨졌는지 표시 + 자동 수정 제안
- 수동 점검에서 사용자가 reject: 사용자가 적은 사유를 `copywriter`에 피드백으로 전달해 재생성
