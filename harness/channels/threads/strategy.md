# Threads Strategy

Meta Threads(@threads) 채널 마케팅 전략. 이 문서는 카피라이팅 단계(`copywriter` 서브에이전트, Phase 3)의 필수 입력이다.

## 한 줄 요약
**대화·관점·짧은 통찰** 중심. 광고 카피처럼 들리면 즉시 외면당한다. 사람 목소리로 쓰되, 회사 톤은 유지.

## 채널 특성

| 항목 | 내용 |
|------|------|
| 글자 수 | 본문 500자 이내 권장 (한국어 기준 ~250자), 최대 500자 |
| 미디어 | 이미지 최대 10장, 영상 5분 이내 |
| 첫 줄 | **첫 1~2줄에서 스크롤을 멈춰야 함** (피드는 본문 일부만 노출) |
| 해시태그 | 1~3개. 남발 금지. (인스타와 다름) |
| 링크 | 허용되지만 노출 페널티 가능성 — 본문 가치를 먼저 |
| 스레드 | 단일 포스트 + 답글 체인으로 시리즈 가능 (`cadence: thread`) |
| API | Threads Graph API 정식 지원 (post, reply, schedule) |

## 콘텐츠 원칙

1. **관점이 먼저, 정보는 나중**
   - "이번 주 데이터 보니 X가 늘었습니다" (X) → "Y는 끝났다고들 하는데, 우리 데이터는 정반대" (O)
2. **첫 줄은 후킹**
   - 질문, 단정, 의외의 숫자, 짧은 장면 묘사
3. **'나/우리' 시점**
   - 1인칭으로 쓰되 회사 페르소나는 일관되게 (예: "샘플페이 팀에서 보니")
4. **마침표보다 줄바꿈**
   - 한 줄 한 호흡. 모바일 가독성 우선
5. **CTA는 본문 마지막 1줄로 짧게**
   - "더 보기 → example.com/x" 정도. 강요 금지

## 금기 (이 채널 한정)

- 인스타식 해시태그 도배 (`#스타트업 #핀테크 #결제 #SMB ...` 5개 이상)
- 광고문구형 어휘 ("지금 신청하세요!!!", "한정 특가")
- 외부 링크를 본문 첫 줄에 배치 (노출 페널티)
- 회사 프로필의 `banned.words / topics / claims` 위반

## 콘텐츠 타입별 권장 형식

| 타입 | 길이 | 미디어 | 시리즈 |
|------|------|--------|--------|
| 인사이트(관점) | 250~400자 | 없음 또는 카드뉴스 1장 | 단일 |
| 케이스 스터디 | 500자 + 카드뉴스 3~5장 | 카드뉴스 | 단일 또는 스레드 3 |
| 제품 업데이트 | 200~300자 | 스크린샷 1~2장 | 단일 |
| 채용 | 300자 | 분위기 사진 또는 카드 1장 | 단일 |
| 시리즈 (티저→본문→마무리) | 각 200~300자 | 통일된 카드뉴스 | 스레드 3 |

## 발행 시간 권장

- 평일 출근(8~9시), 점심(12~13시), 퇴근(18~19시) 윈도우 우선
- B2B 타겟이면 오전 우선, B2C는 저녁 우선

## 회사 프로필 매핑

`copywriter`가 카피 생성 시 다음을 반드시 참조:
- `tone.preset` + `tone.voiceNotes` → 문장 호흡·종결어미
- `tone.sampleSentences` → 모방 학습용 (있는 경우)
- `targetAudience[].painPoints` → 첫 줄 후킹의 재료
- `banned.*` → 사후 검열 (생성 후 필터링)
- `hashtags.always` + `hashtags.pool` (1~3개만 선택)

## 업로드 (Phase 4 — Threads Graph API)

```
1. POST /me/threads (text + media_url) → creation_id
2. POST /me/threads_publish?creation_id=... → 발행
3. (선택) GET /me/threads/{id}/insights → 도달·노출
```

자세한 어댑터 구현은 `publisher/threads/` 참조 (Phase 4).
