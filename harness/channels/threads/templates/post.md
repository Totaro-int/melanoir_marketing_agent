# Threads Post Templates

`copywriter` 서브에이전트가 사용할 카피 템플릿. 각 템플릿은 **구조 가이드**일 뿐, 그대로 채워 넣지 말 것 — 회사 프로필의 톤으로 재해석한다.

---

## T1. 인사이트형 (단일, 250~400자)

```
{후킹 1줄: 의외의 단정 / 질문 / 짧은 숫자}

{관점 2~4줄: 왜 그렇게 보는지, 우리 경험 또는 데이터}

{전환 1줄: 그래서 무엇을 의미하는가}

{CTA 1줄: 선택}
```

**예시 (샘플페이 프로필 기준)**:
```
"PG 통합은 며칠이면 끝난다"는 말, 우리는 안 믿습니다.

지난 분기 50개 SMB를 옮겨보니
평균 5.2일이 1.8일로 줄었습니다.
줄어든 4일은 결국 정산 주기였습니다.

"통합"이 아니라 "정산"이 진짜 병목이었던 셈.

example.com/case
```

---

## T2. 케이스 스터디형 (단일 + 카드뉴스 3~5장)

```
{후킹: 결과 숫자 1줄}

{배경 1~2줄: 어떤 회사·어떤 상황}

{방법 1~2줄: 무엇을 바꿨는가}

{결과 1~2줄: 숫자 중심}

(카드뉴스: 타임라인 / 비포-애프터 / 핵심 지표)
```

---

## T3. 제품 업데이트형 (200~300자)

```
{기능 한 줄}

{이게 왜 필요했는지 1~2줄: 사용자 통점 인용}

{무엇이 달라지는지 1~2줄}

{도큐먼트/체인지로그 링크 1줄}
```

스크린샷 1~2장. 마케팅 카피 톤 금지, **릴리스 노트 톤**이 더 잘 받음.

---

## T4. 채용형 (~300자)

```
{직무 한 줄: 우리는 _____를 찾습니다}

{왜 지금 필요한가 1~2줄: 회사가 뭘 만들고 있고 어디서 막혔는지}

{어떤 사람과 일하게 되는가 1~2줄}

{지원 링크 1줄}
```

분위기 사진 1장 (팀·사무실), 또는 로고 카드 1장.

---

## T5. 시리즈 (스레드 3편: 티저 → 본문 → 마무리)

**1편 (티저, 200자)**: 결론을 살짝 던지고 "내일 풀어쓰겠습니다"
**2편 (본문, 300자)**: 이유·데이터·관점
**3편 (마무리, 200자)**: 그래서 우리가 뭐 했는지 + CTA

각 편이 **단독으로도** 가치가 있어야 함 — 스레드를 안 펴는 사용자가 다수.

---

## 채우기 규칙 (copywriter용)

1. `{...}` 자리는 회사 프로필의 톤으로 재작성. 템플릿 문장을 그대로 두지 말 것.
2. `tone.sampleSentences`가 있으면 그 호흡을 따라할 것.
3. 첫 줄은 **80자 이내** (피드 노출 컷).
4. 해시태그는 본문 끝에 줄바꿈 후, 1~3개만.
5. 카드뉴스 프롬프트는 `image-director`에게 위임 (별도 산출물).
