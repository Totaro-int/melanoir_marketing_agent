# Brunch 발행 전 체크리스트

> 공통 점검은 [`../blog/checklist.md`](../blog/checklist.md) 참조.  
> 본 문서는 브런치 매체 특수 점검만.

## 자동 점검 (브런치 추가)

### 메타·태그
- [ ] **매거진 지정** 필수 (`brunch.magazineId` 필드)
- [ ] 태그 5개 한도
- [ ] 제목 길이 자유 (긴 시적 제목 OK)

### 톤
- [ ] **광고 어휘 0건** (브런치는 즉시 거부)
- [ ] "방법", "정리", "가이드" 같은 SEO 키워드 X (대신 시적·문학적)
- [ ] 1인칭 사용 ("저는", "우리는") — 객관 보고체보다 우선

### 사진
- [ ] 헤더 사진 1장 (필수)
- [ ] 본문 사진 1~3장 (인물 OK)
- [ ] 사진 저작권 명시 (Unsplash credit OK)

## 수동 점검 (브런치 추가)

- [ ] 매거진 일관성? (이전 글들과 같은 톤·주제)
- [ ] 에디토리얼 매거진 톤인가? (보고서 X)
- [ ] 시적·관찰적 제목? (SEO 키워드 도배 X)
- [ ] 마지막 disclaimer 없는가? (브런치는 작가의 글)
- [ ] 사진이 본문 호흡과 맞는가?

## 실패 시 행동

- 자동 실패: 어느 룰 깨졌는지 표시 + 자동 수정 제안
- 수동 reject: copywriter 재생성 (브런치 톤 강조 — 시적·관찰)
