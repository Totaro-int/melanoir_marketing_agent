# Brunch Strategy

> ⚠️ **공통 정보성 블로그 가이드는 [`../blog/strategy.md`](../blog/strategy.md) 참조.**  
> 본 문서는 브런치 매체 특수 차이만.

## 한 줄 요약
**에디토리얼 매거진 톤 + 사진 풍부 + 길이 자유**.  
다른 블로그와 달리 광고·SEO 어조 X. 카카오 + 구글 SEO. 인물 사진·문학적 표현 OK.

## 브런치 매체 특수 차이

| 항목 | 값 / 정책 |
|------|----------|
| 검색 알고리즘 | 카카오 + 구글 SEO + 브런치 자체 추천 (매거진·작가) |
| 본문 길이 권장 | **1,500~3,000자** (자유, 길어도 OK) |
| 제목 권장 | 자유 (긴 시적 제목 OK) |
| 태그 한도 | **5개** (long-tail 위주) |
| 매거진 | 작가 매거진 단위로 묶임 (필수) |
| 이미지 | **풍부 사용 OK** — Unsplash 통합, 헤더 + 본문 1~3장 |
| 인물 사진 | **OK** (에디토리얼이라 자연스럽게) |
| 발행 빈도 | 매주 1~2회 (정기성 ↑ → 작가 추천 ↑) |
| 상업성 톤 | **약함** — 홍보·광고 톤 즉시 거부됨 |
| B2B 적합도 | ★★ (B2C·문화·라이프스타일·인사이트 우세) |

## 브런치 SEO 추가 디테일

### 매거진 우선
- 글 발행 시 **매거진** 지정 필수 (작가 카테고리 = 매거진)
- 매거진 일관성이 작가 추천 알고리즘에 핵심
- 한 매거진 안에 같은 톤·주제로 묶기

### 에디토리얼 톤
- "방법" "정리" "가이드" 같은 SEO 키워드 X
- 시적·문학적 제목 OK ("어느 봄날의 정산일", "OEM을 찾는 일은 결국 사람을 찾는 일이다" 류)
- 1인칭 ("저는…", "우리는…") + 경험·관찰 우선

### 사진의 비중
- 헤더 사진 1장 (Unsplash 무료 통합)
- 본문 중간 1~3장 (분위기·인물·풍경 OK)
- 사진 저작권 명시 (Unsplash credit 자동 처리됨)

### 카카오 추천
- 댓글·공유 신호 → 카카오 다음 메인 노출 가능성
- 작가 팔로워가 많을수록 추천 ↑

## 발행 (Phase 4 — 공식 API 없음)

브런치는 공식 발행 API가 없음. 다음 옵션:

### 옵션 1: browser-publish (권장)
`harness/bin/browser-publish.mjs` (origin pull로 들어옴) 활용:
1. brunch.co.kr 로그인 (Kakao SSO)
2. "글쓰기" 페이지 navigate
3. 제목·매거진·본문 입력
4. 이미지 첨부 (Unsplash 검색 또는 업로드)
5. 발행 직전 멈춤 → 사용자 1번 클릭

### 옵션 2: dry-run
`--dry-run` 플래그 시 컴포저까지만 채우고 게시 직전에 멈춤 (본문·이미지 확인용).

발행: browser-publish (크롬 쿠키) — `node harness/bin/browser-publish.mjs <slug> --channel=brunch --attach --pre-publish`

## 회사 프로필 매핑

`copywriter`:
- `tone.preset` → 에디토리얼 톤으로 변환 (정보형 → 시적·관찰)
- 1인칭 ("우리는...") + 경험·관찰 우선
- 키워드 stuffing X (자연 분포)
- 마지막 disclaimer 절대 X (브런치는 작가의 글이라는 전제)
- 태그 5개 (long-tail)

`image-director`:
- 6-component 공식 적용
- 채널 = brunch → **인물 OK** (`allowPeople: true`)
- 사진 풍부 (헤더 + 본문 1~3장)
- aestheticDirection "editorial" 또는 "organic" 우선
