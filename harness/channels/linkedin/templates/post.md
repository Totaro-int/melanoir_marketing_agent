# LinkedIn Post Templates

## L1. 인사이트형 (800~1200자)

```
{후킹 1줄: 한 단어 또는 숫자, 마침표로 끊기}

{후킹 2줄: 의미 풀이}

{후킹 3줄: 본문 갈고리}

{본문 단락 1: 배경/맥락 — 2~3줄}

{본문 단락 2: 우리가 본 것 — 2~3줄}

{본문 단락 3: 그래서 무엇이 달라지는가 — 2~3줄}

{마지막 1줄: 댓글 유도 질문}

{해시태그 3~5개}
```

**예시 (업플로우 기준)**:
```
1.8일.

지난해 5.2일이던 정산 주기가 1.8일이 됐다.

방법은 의외로 단순했다.

대부분의 SMB 결제 통합은 "PG 연동"에서 끝난다고 본다.
하지만 50개 가맹점을 옮겨보니 진짜 병목은 그 다음, 정산 단계였다.

PG·간편결제·계좌이체의 정산 주기를 한 줄로 정규화하니
운영팀이 매주 들이던 8시간이 1시간으로 줄었다.

결국 "통합"의 진짜 가치는 데이터가 아니라 운영 시간이다.

정산 주기를 줄여 달라진 건 무엇이었나요? 댓글로 듣고 싶습니다.

#fintech #B2BSaaS #정산자동화 #SMB
```

## L2. 케이스 스터디형 (1200~1800자 + 캐러셀 6~9장)

```
{후킹 3줄: 결과 숫자 + 회사명 익명/공개 + 한 마디 평}

{Before: 어떤 상황이었나 — 4~6줄}

{What we did: 무엇을 바꿨나 — 4~6줄, 의사결정 근거 포함}

{After: 결과 — 4~6줄, 숫자 위주}

{Lesson: 다른 회사가 가져갈 만한 교훈 1~2줄}

{질문 1줄}

{해시태그}
```

캐러셀 구성: ①후킹 ②Before ③What ④After 1 ⑤After 2 ⑥Lesson ⑦CTA

## L3. 제품/회사 업데이트형 (600~900자)

```
{무엇이 새로워졌는지 1줄}

{왜 만들었나 — 사용자 통점 인용 2~3줄}

{어떻게 다른가 — 2~3줄}

{지금 무엇을 할 수 있나 — 1~2줄}

{문서/체인지로그 — "👇 댓글에 링크" 표기}

{질문}

{해시태그}
```

## L4. 채용형 (800~1200자)

```
{직무 한 줄: "우리는 ___을 찾습니다"}

{왜 지금 — 회사가 무엇을 만들고 있고 어디서 막혔는지 3~5줄}

{이상적 동료 — 기술 1~2줄 + 태도 1~2줄}

{함께할 사람들 — 팀 분위기 1~2줄}

{프로세스 한 줄}

{👇 댓글에 채용 페이지}

{질문 — "어떤 점이 가장 궁금하세요?"}

{해시태그}
```

---

## 채우기 규칙

1. 첫 3줄 합산 220자 이내. 매 줄을 마침표로 끊고 줄바꿈.
2. `tone.preset`이 친근/위트라도 LinkedIn에선 한 단계 professional로 보정.
3. 외부 링크는 본문에 넣지 말 것 — 마지막에 "👇 댓글에 링크" 표기 후 첫 댓글 자동 생성.
4. 마지막은 반드시 질문으로 끝내기 (댓글 유도).
5. 캐러셀 텍스트는 모바일 가독성 우선 (한 슬라이드에 두 문장 이상 금지 권장).
