# Blog (정보성 글) 통합 발행 전 체크리스트

> 모든 블로그 매체(naver-blog / tistory / brunch) 공통.  
> 매체별 추가 체크는 매체별 channel 폴더의 checklist.md 참조.

## 자동 점검 (Phase 3 게이트가 차단)

### 모드 (`blogMode` 필드 — 미지정 시 brief.goal → 자동 매핑)
- [ ] `blogMode` 가 5개 중 하나: `default` / `rcon` / `ai-briefing` / `home-plate` / `insight-edge`
- [ ] 모드별 분량·구조 규칙 충족 (아래 모드별 자동 점검 표 참조)

### 모드별 자동 점검 표

| 룰 | default | rcon | ai-briefing | home-plate | insight-edge |
|---|---|---|---|---|---|
| 본문 분량 (공백 포함) | ≥ 2,000 | ≥ 2,000 | ≥ 2,500 | 1,500~2,000 | 2,000~2,500 |
| 제목 길이 | 25~35자 | 자유 | 25~35자 | 22~25자 | 25~30자 |
| H2 최소 개수 | 4 | **3 필수** | 4 | 3 | 4 |
| 표(Table) | 1개 이상 | 선택 | **필수** | 선택 | 선택 |
| 인용구(`>`) | 선택 | 선택 | 선택 | **필수 1개+** | **필수 1개+** |
| FAQ 섹션 | 옵션 | 옵션 | **필수** | 보통 비활성 | 옵션 |
| 짧은 문단 (3~4줄) | 권장 | **필수** | 권장 | **필수** | 권장 |
| 시의성 어휘 (최신/오늘/2026) | - | **필수** | - | - | - |
| 1인칭 경험담 톤 | 권장 | 권장 | 약함 | **필수** | 권장 |
| Pain Point 첫 30% | - | - | - | - | **필수** |

### 본문 (모드 무관 공통)
- [ ] **H1은 제목 전용** (본문에 H1 절대 사용 X)
- [ ] **본문 첫 줄에 H1(`#`) 절대 금지** — 제목은 별도 필드
- [ ] **마크다운 구분선 (`---`, `***`) 절대 금지**
- [ ] H2/H3 텍스트 자체에 `#` 기호 포함 X
- [ ] 핵심 키워드(첫 단락 + 본문)에 자연 분포 3~7회
- [ ] **첫 단락(150자 내)**에 핵심 키워드 1개 이상 포함
- [ ] **첫 단락에 외부 링크 없음** (SEO 페널티)
- [ ] `banned.words` 미포함
- [ ] `banned.claims` 미포함 ("최고의", "1위", "유일한", "100%" 등)
- [ ] `banned.topics` 미포함

### AI 클리셰 / 금지 패턴 (5 모드 공통 — stealthMode 활성화 시 더 엄격)
- [ ] "다양한 방면에서" 미사용
- [ ] "함께 알아보겠습니다" 미사용
- [ ] "결론적으로" / "마지막으로" 미사용 (도입·맺음 클리셰)
- [ ] "~에 대해 알아보겠습니다" 미사용 → 바로 본론
- [ ] "장점 N가지" 식 정보 요약형 H2 → 감정/질문형 H2 로 대체
- [ ] "이상으로 마치겠습니다" 미사용 → 열린 질문 / 캐주얼 작별
- [ ] "중요한 것은 바로 이것입니다" 미사용 → 직접 설명

### 제목
- [ ] 제목 길이 ≤ 60자 (네이버 검색 결과 노출 컷)
- [ ] 제목 권장 25~30자 (한글 검색 미리보기 안전)
- [ ] 제목에 **핵심 키워드 1개 이상 앞부분(앞 25자 이내)** 배치
- [ ] 제목에 광고 어휘 없음 (느낌표 금지, "지금" 단독 금지, "최고/1위/유일한" 금지)
- [ ] 제목 키워드 stuffing 없음 (같은 키워드 2회+ 반복 X)

### 키워드 분포
- [ ] 메인 키워드 본문 3~7회 (stuffing 8회+ 금지)
- [ ] 연관 키워드 2~4종 자연 분포
- [ ] 의미 없는 특수문자 반복 없음

### 이미지·링크
- [ ] 헤더 이미지 1장 (썸네일·SEO·소셜 미리보기용)
- [ ] 본문 이미지 1~3장 권장 (H2 사이마다)
- [ ] 모든 이미지 ALT 텍스트 있음
- [ ] 내부 링크 2~4개 (관련 글 연결)
- [ ] 외부 링크 0~2개 (신뢰 사이트만, 첫 단락 X)

### 태그·메타 (매체별 한도)
- [ ] 태그 long-tail 위주 (단순 1단어 키워드 X)
- [ ] (광고 분류 시) 본문 시작 또는 끝에 광고 표시

## 수동 점검 (사람이 확인)

### SEO·검색 의도
- [ ] 사용자가 무엇을 검색했을 때 이 글이 답이 되는지 명확한가?
- [ ] 제목·첫 단락이 그 검색 의도와 일치하는가?
- [ ] 키워드 stuffing처럼 어색한 부분 없는가?

### 본문
- [ ] 회사 톤(`tone.voiceNotes`)과 일치하는가? (정보형으로 약간 객관화 OK)
- [ ] 광고 외침이 아닌 정보 전달인가?
- [ ] 사실 확인 — 숫자·인용·날짜 정확성
- [ ] H2 각 섹션 첫 줄에 결론·요약이 있는가? (스캔 읽기 최적화)
- [ ] CTA가 본문 끝에 자연스럽게 들어갔는가? (강요 X)

### 가독성
- [ ] 짧은 문단 (3~5줄 단위)인가?
- [ ] 이미지가 본문 흐름을 끊지 않고 받쳐주는가?
- [ ] 끝까지 읽히는가? (개인 경험담·예시·번호 매김 활용)

## 실패 시 행동

- 자동 점검 실패: 어느 룰이 깨졌는지 표시 + 자동 수정 제안
- 수동 점검 reject: 사용자 피드백을 `copywriter`에 전달해 재생성
