# 블로그 본문 스타일 가이드 (relate.kr 톤)

> copywriter / brand-guardian / image-director 에이전트가 모든 블로그 본문 작성 시 자동 적용.
> 레퍼런스: https://www.relate.kr/blog/relate-mcp-sales-automation-usecases/

## 1. 톤 — 정갈한 정중체

- 정중체 통일 — `~합니다`, `~할 수 있습니다`, `~됩니다`
- 친근체 금지 — `~해요`, `~거든요`, `~예요`, `~잖아요`
- 영업 톤 자제 — "지금 바로 만나보세요", "놓치지 마세요" 같은 표현 X
- 단정 자제 — "최고", "유일한", "혁신적" 같은 추상 형용사 X
- 비즈니스 정보 전달 + 신뢰감 — 학술 인용 / 정량 수치 / 비포-애프터 비교

## 2. 구조 — 한 단락 한 메시지

- 단락 길이 — 평균 2-3 문장. 가끔 1 문장. 7 문장 이상 단락 금지.
- 한 단락 한 주장 — 여러 사실을 한 단락에 욱여넣지 않음.
- 도입부 — 4-5줄. 짧고 직설적. 무엇을 / 왜 / 어떻게 의 답을 첫 단락에 명시.
- 마무리 — 추상적 감상 X. 사실 진술 또는 체크리스트.

## 3. 시각 요소 — 절제

- 이모지 — **0개**. 본문 어디에도 사용 금지.
- bold (`**텍스트**`) — 최소화. 핵심 명제 1-3회만. 또는 0개.
- italic — 사용 X (특수 인용 외).
- 헤딩 — H2 (`##`) 와 H3 (`###`) 만 사용. H4 이하 X.
- 인용구 (`>`) — 실제 발화 또는 핵심 명제만. 감탄/수사 인용 X.

## 4. 리스트 — 키워드 — 한 줄 설명 패턴

```
- 키워드 — 한 줄 설명입니다.
- 다른 키워드 — 다른 한 줄 설명입니다.
```

대시 (`—` 또는 `-`) 로 키워드와 설명 구분. 한 항목 = 한 줄. 두 줄 이상 항목은 별도 단락으로.

## 5. 정량 표현

- 추상 수식 X — "효율적", "다양한", "혁신적", "획기적", "강력한"
- 정량 수치 그대로 — `SPF 2.5`, `λc 388 nm`, `UVA/UVB 0.87`, `4,086자`, `3-4시간 → 30분`
- 비포-애프터 강조 — 변화 수치를 제목/도입부에 박음

## 6. FAQ 표준 형식

```
## 자주 묻는 질문

Q.1 질문 텍스트?

A. 답변 텍스트입니다.

Q.2 다음 질문?

A. 다음 답변입니다.
```

- 각 질문 앞 `Q.N` 번호
- 답변 앞 `A.` 접두
- Q/A 사이 빈 줄
- 각 FAQ 항목 사이 빈 줄
- bold 강조 X (질문도 답변도 평문)

## 7. 표 (네이버 블로그)

- 마크다운 표 사용 OK — 4-5 컬럼, 4-6 행 권장
- 표 위에 캡션 1줄 ("핵심만 표로 정리하면 다음과 같습니다.")
- 표 안에 이모지 X
- 셀 내용 — 키워드 또는 짧은 구 (한 문장 X)

## 8. 인용구 — 핵심 명제 강조

```
> 정량 수치가 천연이라는 표현보다 선행 기준입니다.
```

- 단락 1개당 인용구 1개 이하
- 글 전체 인용구 2-3개 이하
- 인용구 안에 이모지 X, 감탄사 X

## 9. 마무리 — 체크리스트 패턴

```
오늘 정리를 짧게 묶으면 다음과 같습니다.

- 광역 차단 데이터(임계파장·UVA/UVB 비율)가 공개돼 있는가
- 항산화 활성 수치가 정량으로 명시돼 있는가
- 세포 안전성 시험 데이터가 있는가
- 단독 차단인지 보조·부스터 용도인지 명확한가
- 근거가 검증된 논문이나 공인기관 시험인가
```

체크리스트 5개 이하. 각 항목 한 줄.

## 10. 광고법 (한국 화장품·반영구·식약처)

> **brand-guardian.mjs 가 자동 검증** — 다음 패턴이 본문에 있으면 자동 block (발행 거부).
> 사용자 별도 등록 불필요.

mustExclude — 본문에 절대 사용 X (block 자동):
- 치료, 의학적 효능, 효과 보장, 부작용 없음
- `100% 안전/순수/천연/자연`
- 최고의, 유일한, 완벽한, 기적의, 즉각적인 효과, 영구적인

영업 톤 자제 (warn 자동):
- "지금 바로 시작/구매/만나", "놓치지 마세요"
- "곧 마감", "한정 수량", "마감 임박"
- "클릭만 하면 즉시"

브랜드 직접 영업 — 글의 가운데가 아닌 한계 / FAQ 섹션에 1회 자연 노출 (정보성 글일 때).
직접 영업 글 (제품 소개) 일 때만 brand-guardian 의 광고 표기 검수 통과.

자료 인용 — sourceMaterials 있으면 본문 끝 `## 참고` 섹션 의무 (warn 자동).

## 11. 채널별 분량

- 네이버 블로그 — 2,000-4,500자 (C-Rank / D.I.A.+ 평균)
- 인스타그램 캡션 — 300-540자
- Threads — 200-300자 (hook 위주)
- LinkedIn — 500-1,200자 (B2B 인사이트 톤)
- Tistory — 1,500-3,500자 (긴 글 SEO)
- 브런치 — 2,000-5,000자 (에세이 톤 OK)

## 12. 채널별 톤 차이

- 네이버 블로그 — 정중체 + 정량 수치 강조 + AEO 친화
- 인스타 캡션 — 정중체 유지 + 두괄식 (첫 줄에 hook)
- Threads — 짧은 정중체 + 한 메시지
- LinkedIn — B2B 정중체 + 영문 키워드 자연스럽게 (예: "MoQ", "OEM", "B2B")
- Tistory — 같은 정중체. 단 SEO 키워드 본문 초반 배치
- 브런치 — 정중체 유지 + 단락 살짝 길게 (3-5 문장) 허용

## 13. mustInclude / mustExclude 우선 — brief.yaml

각 캠페인 brief.yaml 의 `constraints.mustInclude`, `constraints.mustExclude` 는 본 가이드보다 우선.
캠페인 특수 요구가 가이드와 충돌 시 brief 따름.
