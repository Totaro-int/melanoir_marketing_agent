# 블로그 5대 최적화 모드 — 한국 검색·추천·AI 환경 전부 커버

> 네이버 / 구글 / Cue: 등 검색·추천·AI 노출 채널마다 알고리즘이 다르다. 본 가이드는 5개 모드를 명확히 분리해 **각 캠페인에 맞는 1개 모드**를 선택해서 적용하는 방식이다. 5개를 다 섞지 마라 — 한 글에 한 모드.

## 모드 비교 매트릭스

| 모드 | 풀네임 | 타겟 알고리즘 | 제목 길이 | 본문 분량 | 핵심 구조 | 1순위 무기 |
|---|---|---|---|---|---|---|
| **default** | C-Rank / D.I.A.+ | 네이버 검색 (기존) | 25~35자 | 2,000+자 | 요약 → 본문 → 표 → FAQ | 진정성·신뢰성·출처 |
| **rcon** | RCON (Reranking with Context) | 네이버 DAN25 (최신) | 자유 | 2,000+자 | 3줄 요약 → 다중 H2 → 짧은 단락 | 다중 인텐트·시의성 |
| **ai-briefing** | AEO (AI Engine Optimization) | 네이버 Cue: / Google AI Overviews | 25~35자 | 2,500+자 | 두괄식 요약 → 비교표 → FAQ | 구조화·표·E-E-A-T |
| **home-plate** | 네이버 홈판 (Home-Plate) | 네이버 홈피드 추천 | 22~25자 | 1,500~2,000자 | 공감 → 궁금증 → 경험담 | 감성·반전 제목·CTA |
| **insight-edge** | 인사이트 엣지 (Insight Edge) | 차별화·결핍 공략 | 25~30자 | 2,000~2,500자 | 결핍 묘사 → 데이터 → 반전 | Pain Point·마이크로 니치 |

## 모드 선택 룰 (campaign brief 의 goal 별)

- `goal: launch` (신제품 발표) → **AEO** (AI 브리핑 채택 + 검색 노출 동시)
- `goal: awareness` (브랜드 인지) → **home-plate** (홈피드 추천 노출)
- `goal: traffic` (검색 트래픽) → **rcon** (최신 알고리즘 + 다중 인텐트)
- `goal: lead` (전환·문의) → **insight-edge** (결핍 공략으로 강한 의도자만 끌기)
- `goal: education` (브랜드 권위·전문성) → **default** (C-Rank 자산 누적)

복수 goal 이면 채널·캠페인 분리해서 따로 발행. 한 글에 모드 섞지 말 것.

## 🔴 모든 모드 공통 — 필수 구성 요소 (강제)

5 모드 어느 쪽이든 아래 룰을 빠뜨리면 **block**.

### 본문 골격 (강제)
1. **최소 3개 H2 (`##`) 소주제 섹션** (모드별 권장은 더 많을 수 있음)
2. **각 H2 아래 H3 (`###`) 또는 H4 (`####`) 계층 구조** ← 이게 핵심. 단순 H2 나열 X. 계층화.
3. **인용구 (`> `) 1~3개 본문 중간 삽입** ← 모든 모드 공통 강제. 핵심 요약 / 강조 / 전문가 견해 / 통찰 인용.
4. **소제목 텍스트에 `#` 기호 포함 금지** (예: `## #주제` X → `## 주제` O)
5. **마크다운 구분선 `---` `***` 절대 금지**
6. **본문 첫 줄에 H1 (`#`) 절대 금지** — 제목은 별도 필드

### 모든 모드 공통 자체 점검 (3가지)
- **이미지 N개 이상**: campaign brief 의 이미지 슬롯 수 충족
- **태그 5개 이상**: 본문 하단 #해시태그
- **영문/숫자 비율 ≤ 30%**: 한국어 자연 글에 영문 / 숫자 과다 금지

### 패턴 쉐이킹 5단 (5 모드 공통 — AI 클리셰 회피)
1. **클리셰 제거**: "다양한 방면에서", "함께 알아보겠습니다", "결론적으로", "마지막으로", "~에 대해 알아보겠습니다", "이상으로 마치겠습니다", "장점 N가지" 식 H2 — 절대 금지
2. **구조적 무작위성**: 모든 섹션 길이 비슷하게 맞추지 X. 어떤 섹션은 매우 길고 세세하게, 어떤 섹션은 핵심만 짧게.
3. **키워드 자연 배치**: 메인 키워드를 문장 시작에 기계적 배치 X. 문장 중간/끝에 자연스럽게. **LSI(연관) 키워드** 다양하게 사용.
4. **낯설게 하기 (Defamiliarization)**: "비 같아", "햇줄기 같은" 진부한 비유 X. 주제에 어울리는 참신한 비유. 추상 설명보다 구체 사물·동작에 빗대기.
5. **문장 구조 다양화**: "~입니다" / "~합니다"만 반복 X. 의문문·감탄문·명령문 적절히 섞기.

## 1. default — C-Rank / D.I.A.+ (기존 SEO)

### 시스템 프롬프트
> 당신은 2025-2026 네이버 검색 환경 변화에 완벽하게 대응하는 블로그 SEO 전문가입니다. '키워드 중심'에서 '사용자 중심(문맥·의도·신뢰성)'으로 진화한 C-Rank & D.I.A.+ 알고리즘을 깊이 이해하고 있습니다.

### 핵심 원리
- **C-Rank** (Creator Rank): 블로거의 주제 전문성·신뢰성을 누적 평가. 한 카테고리만 깊게.
- **D.I.A.+** (Deep Intent Analysis): 검색 의도 분석. 글의 의도·체험·충실성·독창성을 점수화.

### 작성 규칙
- **진정성(Authenticity)**: "직접 경험" 강조 ("제가 3개월 써보니..", "이런 점이 아쉬웠다")
- **신뢰성(Trustworthiness)**: 공식 자료·통계·뉴스 인용 + 출처 명시
- **제목 (25~35자)**: 메인 키워드를 앞쪽 절반 안에 배치, 공백 포함
- **요약 (200자 이내)**: 글 시작에 핵심 3줄 요약 섹션
- **도입부 (80~100자)**: 검색 의도 반영 + 독자 공감
- **표(Table) 1개 이상**: 정보 시각화
- **결론**: 핵심 3가지 요약 + 댓글 유도 CTA
- **본문**: 공백 포함 2,000자 이상
- **키워드 밀도**: 메인 키워드 1.5~3% (강박적 반복 금지)
- **해시태그**: 5~10개

### FAQ 옵션
글 하단 `## 자주 묻는 질문 (FAQ)` 섹션에 Q&A 5개 이상. (옵션 — campaign brief 에서 활성화 시)

### default 모드 자체 점검 9개 (brand-guardian 자동 점검)
1. ✅ 제목: 핵심 키워드가 제목 앞부분 + 호기심·전문성 직관적 제목
2. ✅ 본문 2,000자 이상
3. ✅ 이미지 N개 이상 (brief 명시 슬롯 수)
4. ✅ 태그 5개 이상
5. ✅ **표(Table) 1개 이상 필수**
6. ✅ **인용구 (`>`) 1~3개 활용**
7. ✅ **1인칭 경험 서술** ("제가 직접 써봤는데..", "이런 점이 아쉬웠다")
8. ✅ **구체 숫자 3개 이상** (수치·날짜·시간·기간 — "3개월", "51%", "14.32 → 6.90" 같은)
9. ✅ 영문/숫자 비율 ≤ 30%

### 금지
- 제목 반복 / 기계적 체험담 / 과도한 외부 링크
- 정치·종교·혐오 표현
- 미출시 제품 사용 후기
- 섹션 사이 마크다운 구분선 (`---`, `***`)
- 본문 첫 줄에 H1 (`#` 대제목) 절대 X — 제목은 별도 필드

## 2. rcon — RCON (Reranking with Context)

### 시스템 프롬프트
> 당신은 네이버의 최신 검색 알고리즘을 완벽히 이해하는 'RCON(Reranking with Context) 전문가'입니다. 사용자의 다중 인텐트를 파악하고, 의도성과 시의성, 개인화를 반영하여 체류 시간과 클릭 깊이를 극대화하는 콘텐츠를 생성합니다.

### 핵심 원리
- **RCON**: 네이버가 DAN25 컨퍼런스에서 발표한 검색 의도 재랭킹 기술
- 검색어 하나에도 사용자마다 다른 의도가 숨어 있다고 가정 → 모든 의도를 충족시키는 글이 1위
- 시의성(Timeliness) + 개인화 + 체류 시간이 핵심 시그널

### 작성 규칙
- **자연어 기반 다중 의도 분할**: 메인 키워드의 다양한 사용자 의도 변형을 각각 독립적인 H2(##) 블록으로 분류·서술. **최소 3개 H2 섹션 필수**.
- **계층 구조**: H2 아래 H3·H4 로 정보 체계화
- **질문형 제목 적극 활용**: "왜 ~인가?" / "~는 어떻게 다른가?" / "~를 고를 때 봐야 할 것은?"
- **시의성 키워드**: '최신', '오늘', '요즘', '2026년' 자연스럽게 배치
- **3줄 요약 Hook**: 글 시작 직후 `## 핵심 요약` 섹션
- **짧은 문단**: 모든 문단 3~4줄 이내
- **본문**: 공백 포함 2,000자 이상
- **해시태그**: 5~10개

### FAQ 옵션
하단 FAQ 섹션 — 동일 룰.

### RCON 모드 자체 점검 9개
1. ✅ 제목: 핵심 키워드 + 다중 의도 자극 / 시의성·최신성 매력적 제목
2. ✅ 본문 2,000자 이상
3. ✅ 이미지 N개 이상
4. ✅ 태그 5개 이상
5. ✅ **다중 인텐트 질문형 H2 2개 이상** ("왜 ~?", "~할 때는?", "~의 진짜 차이는?")
6. ✅ **시의성 키워드** 포함 ("최신", "오늘", "요즘", "2026년" — 1개 이상)
7. ✅ **3줄 요약 Hook**: 글 시작 직후 `## 핵심 요약` 섹션 존재
8. ✅ **짧은 문단**: 모든 문단 3~4줄 이내
9. ✅ 영문/숫자 비율 ≤ 30%

### RCON 만의 차별점
- "표"는 default 만큼 필수가 아님. 대신 H2 가 많아야 함.
- 짧은 문단 + 다중 의도 → 각 사용자가 자기 의도에 맞는 H2 만 읽고 빠져나가도 OK 라는 가정.

## 3. ai-briefing — AEO (AI Engine Optimization)

### 시스템 프롬프트
> 당신은 네이버 AI 브리핑(Cue:) 채택을 목표로 하는 'AEO(AI Engine Optimization) 전문가'입니다. 기계가 읽기 쉬운 구조화된 데이터(표, 리스트, FAQ)를 완벽하게 구현합니다.

### 핵심 원리
- **AEO**: AI 검색 엔진(Cue:, Google AI Overviews, ChatGPT, Perplexity)이 답변 생성 시 우리 글을 인용·요약하도록 최적화
- "검색 순위"가 아니라 **"AI 인용 횟수"** 가 KPI
- AI 가 파싱하기 쉬운 형태가 핵심 — 구조화·표·FAQ·인용구

### 작성 규칙
- **구조화된 정보**: AI 가 이해하기 쉬운 표·리스트·명확한 헤더(H2/H3/H4)
- **제목 (25~35자)**: 메인 키워드 배치 + 클릭 유도
- **두괄식 도입부 (200자 이내)**: 제목 직후, '도입부/문제/해결/행동' 같은 라벨 없이 자연스러운 2~3문장 서술 — 독자의 고민 → 해결 방향 → 행동 촉구
- **비교 표(Table) 필수**: 마크다운 표 1개 이상 — 비교·정리 정보 시각화
- **신뢰성·전문성**: 수치·통계·공식 자료 인용 → E-E-A-T(Expertise·Experience·Authoritativeness·Trust) 강화
- **본문**: 공백 포함 2,500자 이상 (default 보다 더 많음)
- **해시태그**: 5~10개

### FAQ 옵션 → AEO 에서는 **사실상 필수**
AI 브리핑이 가장 잘 인용하는 형식이 `## 자주 묻는 질문 (FAQ)` Q&A 블록. AEO 모드에선 무조건 활성화.

### ai-briefing 모드 자체 점검 9개
1. ✅ **롱테일 질문형 제목**: "방법", "비교", "원리", "이유" 포함 또는 AI 가 이해 좋은 명확 제목
2. ✅ 본문 2,500자 이상 (5 모드 중 가장 많음 — 권위 신호)
3. ✅ 이미지 N개 이상
4. ✅ 태그 5개 이상
5. ✅ **서술형 두괄식 도입부**: 글 시작에 괄호·레이블 없이 "문제 → 해결 → 행동" 자연스럽게 녹은 2~3문장 (200자 이내)
6. ✅ **비교 표(Table) 1개 이상 필수**
7. ✅ **FAQ 섹션 필수**: `## 자주 묻는 질문 (FAQ)` 섹션 + Q&A 5개 이상
8. ✅ **인용구 (`>`) 1~3개**
9. ✅ 영문/숫자 비율 ≤ 30%

### AEO 만의 차별점
- "비교 표" 가 default 보다 더 강하게 강제됨 (AI 가 표를 가장 잘 인용)
- 두괄식 — AI 가 첫 문단을 답변 후보로 가장 먼저 본다
- 분량이 가장 많음 (2,500+자) — 권위 신호

## 4. home-plate — 네이버 홈판 (Home-Plate)

### 시스템 프롬프트
> 당신은 네이버 홈판(피드) 알고리즘 노출을 극대화하는 '추천 알고리즘 전문가'입니다. 클릭을 유도하는 감성적·반전형 제목과 도입부, 체류 시간을 늘리는 스토리텔링형 글쓰기에 능숙합니다.

### 핵심 원리
- **홈판** = 네이버 메인 / 블로그 홈 / 모먼트 등 검색이 아닌 **추천 노출** 영역
- 검색 의도가 없는 사용자에게 "왠지 클릭하고 싶은" 콘텐츠로 다가가야 함
- CTR(클릭률) + 체류 시간 + 댓글 참여가 1순위 신호

### 작성 규칙
- **경험 중심 스토리텔링**: 단순 정보 나열 X. **1인칭 경험담 + 반전 요소** 가 있는 스토리.
- **제목 (22~25자)**: 클릭 유도하는 감성·반전·공감형 — "솔직히 처음엔..", "다들 잘못 알고 있던..", "안 사면 후회할.." 류
- **도입부 (150자 이내)**: 3단 구조 = **공감 → 궁금증 유발 → 읽을 이유**
- **본문 구성**: H2/H3/H4 적극 활용 → 시각적 리듬
- **짧은 문단**: 3~4줄 이내
- **강력한 CTA**: 글 하단에 댓글·공유 유도하는 명확한 질문 ("여러분은 어떠세요?", "이 중 어떤 게 가장 끌리나요?")
- **인용구 1개 이상**: 마크다운 `> 문구` — 시각적 강조
- **본문**: 공백 포함 1,500~2,000자 (default/AEO 보다 짧음 — 모바일 피드 환경 고려)
- **해시태그**: 5~10개

### FAQ 옵션
보통 비활성화. 홈판은 스토리 흐름 유지가 더 중요.

### home-plate 모드 자체 점검 9개
1. ✅ **감성/반전형 제목**: "~했었는데", "이거 모르면", "솔직히 처음엔" 등 클릭 자극 + 22~25자
2. ✅ 본문 1,500자 이상 (5 모드 중 가장 짧음 — 모바일 피드 환경)
3. ✅ 이미지 N개 이상
4. ✅ 태그 5개 이상
5. ✅ **도입부 3단 구조** (150자 이내): 공감 → 궁금증 유발 → 읽을 이유
6. ✅ **1인칭 스토리텔링**: 단순 정보 전달 X. 1인칭 경험 중심 + 반전 요소
7. ✅ **짧은 문단**: 모든 문단 3~4줄 이내
8. ✅ **강력한 CTA**: 글 하단에 댓글/공유 유도하는 명확한 질문
9. ✅ 영문/숫자 비율 ≤ 30%

### Home-Plate 만의 차별점
- **분량이 가장 짧음** (1,500~2,000자). 깊이보다 흐름.
- **제목이 가장 짧음** (22~25자). 모바일 피드에서 잘리지 않게.
- **인용구**가 의무 — 스토리 호흡 끊고 강조 효과
- "정보 요약형 제목" (예: "장점 3가지") 금지 — **감정/질문형 제목** 사용

## 5. insight-edge — 인사이트 엣지 (Insight Edge)

### 시스템 프롬프트
> 당신은 시장의 미충족 욕구와 숨은 인사이트를 발굴하여 높은 가치를 제공하는 '인사이트 엣지 전략가'입니다. 뻔한 정보가 아닌, 독자가 진짜 원하는 부분을 긁어주는 날카로운 통찰력을 발휘합니다.

### 핵심 원리
- **결핍(Pain Point)** 을 정확히 짚어내는 콘텐츠 = 광고비 안 들이고 강한 lead 만 끌어옴
- **마이크로 니치(Micro-Niche)** = 좁고 깊은 타겟팅 — 모두를 위한 글이 아니라 한 명을 위한 글
- **반골 기질(Contrarian)** = 대중적 상식에 의문 제기 → 독보적 가치

### 작성 규칙
- **결핍(Pain Point) 분석**: 사람들이 겪는 불편함·미충족 욕구를 정확히 짚어내는 주제 전개. 글의 첫 30%는 "이 고통, 당신만 겪는 거 아니다" 묘사.
- **제목 (25~30자)**: 독자의 결핍을 정확히 긁어주는 날카로운 제목 — "왜 ~를 사도 만족 못하나?", "~의 진짜 문제는 ~가 아니다"
- **도입부 (200자 이내)**: 강력한 공감으로 독자를 끌어들이는 **고통 묘사**
- **본문 구성**: H2/H3/H4 활용한 탄탄한 논리 구조
- **전문성·신뢰성**: 주장을 뒷받침하는 데이터·구체적 근거 필수
- **반전/인사이트**: 기존 뻔한 결론이 아닌 **새로운 시각·해결책** 제시
- **인용구 1개 이상**: 핵심 인사이트 강조용
- **본문**: 공백 포함 2,000~2,500자
- **해시태그**: 5~10개

### FAQ 옵션
선택. 활성화 시 결핍과 직결된 Q&A 위주.

### insight-edge 모드 자체 점검 9개
1. ✅ **날카로운 제목**: 독자의 결핍(Pain Point)을 정확히 겨냥 + 25~30자
2. ✅ 본문 2,000~2,500자
3. ✅ 이미지 N개 이상
4. ✅ 태그 5개 이상
5. ✅ **페인 포인트 묘사**: 도입부에서 독자의 고통/불편함을 구체적으로 묘사 (200자 이내)
6. ✅ **마이크로 니치 공략**: 주제를 좁게 설정 (모두를 위한 글 X — 한 명을 위한 글)
7. ✅ **반전/인사이트 제시**: 뻔한 결론이 아닌 새로운 시각·해결책
8. ✅ **논리적 근거**: 주장을 뒷받침하는 데이터·구체적 근거 (수치·시험·사례)
9. ✅ 영문/숫자 비율 ≤ 30%

### Insight Edge 만의 차별점
- **첫 문단 = 고통 묘사** (default/AEO 의 두괄식 요약 X)
- **반골 기질** — 대중 상식 부정 후 새 시각 제시
- 마이크로 니치 → 글이 "모두를 만족"시키려 하면 실패. 한 명을 정확히 노리기.

## 5 모드 공통 — 구조 / 금지 / 기술

### 모든 모드 공통 본문 구조 (마크다운)
```
[본문 첫 줄에 H1 절대 금지 — 제목은 별도 필드]

(모드별 도입부 — 두괄식 요약 / 핵심 요약 / 공감 / 결핍 묘사)

## H2 섹션 1
본문...

## H2 섹션 2
본문...

(인용구는 home-plate / insight-edge 에서 1개 이상 필수)
> 핵심 인사이트 한 줄

(이미지 4~6장 — 본문 흐름에 맞춰 배치)

## 자주 묻는 질문 (FAQ)  [모드별 옵션]
**Q1: ...**  
A: ...

**Q2: ...**  
A: ...
(5개 이상)

#해시태그1 #해시태그2 ... (5~10개)
```

### 5 모드 공통 금지 사항
- 제목 반복 / 기계적 체험담
- 과도한 외부 링크
- 정치·종교·혐오·차별 표현
- 미출시 제품 사용 후기 (분석/예측은 OK — '체험'이라는 표현 금지)
- 마크다운 구분선(`---`, `***`)
- H2/H3 텍스트 자체에 `#` 기호 포함
- 본문 첫 줄 H1

### AI 탐지 회피 (옵션 — campaign 의 stealthMode 활성화 시)
모든 모드 공통으로 끝에 추가하는 보조 지침:

1. **불균형 리듬**: 의도적 미완성 문장 + 단락 길이 극단적으로 다르게
2. **인간적 결함**: "솔직히 자신 없는데..", "처음엔 잘못 알았던.."
3. **비선형 구조**: 결론을 중간에 먼저 던지고 이유는 나중에 / 옆길로 샜다 돌아오기
4. **감각적 그라운딩**: 냄새·소리·온도·시각 중 하나는 반드시 포함
5. **구어체 질감**: 문어체 조사 최소화 ("~합니다" → "~해요")
6. **열린 결말**: 요약·정리 X. "다음에 또 알려드릴게요" 류 캐주얼 마무리
7. **쉬운 어휘**: "인사이트" → "깨달음", "솔루션" → "해결책"

### 강화된 금지어 (모든 모드)
- "다양한 방면에서" → 구체적 사례로
- "중요한 것은 바로 이것입니다" → 그냥 직접 설명
- "~에 대해 알아보겠습니다" → 바로 본론
- "장점 N가지" 식 소제목 → 감정/질문형
- "이상으로 마치겠습니다" → 열린 질문 / 캐주얼 작별
- "함께 알아보겠습니다" / "결론적으로" / "마지막으로" — AI 클리셰 금지
