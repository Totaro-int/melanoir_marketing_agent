# Naver Blog Post Templates

`copywriter` 서브에이전트가 사용할 블로그 글 템플릿. 각 템플릿은 **구조 가이드**일 뿐, 그대로 채워 넣지 말 것 — 회사 프로필의 톤·SEO 키워드로 재해석한다.

> 네이버 블로그는 검색 매체. **SEO 친화 + 정보 밀도 + 1500자 이상**이 핵심.

---

## N1. 가이드형 (2,000~2,500자, H2 5~6개)

**제목 패턴**: "{핵심 키워드}, {결과 약속} — {차별점}"

**구조**:
```
[제목]
{핵심 키워드를 앞 25자 안에 배치한 명확한 제목}

[도입 단락 — 약 200자]
{문제 환기 + 핵심 키워드 1회 자연 삽입}
{해결 방향 한 줄 미리 제시 (검색 의도 충족)}

[H2-1: 문제 정의 — 약 400자]
{왜 그게 문제인가, 누가 겪는가}

[H2-2: 핵심 솔루션 — 약 500자]
{제품·방법·접근 — 가장 구체적 정보}

[H2-3: 차별점 또는 기능 상세 — 약 400자]
{경쟁 방식과 어떻게 다른가}

[H2-4: 사용 사례 또는 적용 단계 — 약 400자]
{실제 어떤 단계에서 가장 유용한가}

[H2-5: 마무리·CTA — 약 200자]
{N (대상 좁히기) + A (행동 유도, soft)}

[태그 10~15개]
{핵심 키워드 + long-tail 변형}
```

**예시 도입 단락 (cos.totaro 톤)**:
```
해외 바이어가 K뷰티 OEM 후보를 좁히는 데 평균 4시간 이상이 걸린다는 얘기를 자주
듣습니다. 인증 서류는 제조사마다 양식이 다르고, MOQ와 리드타임은 표로 정리하다 보면
한나절이 갑니다. 한 라인 검토에 출장 한 번이 더 붙으면 출시 일정이 그대로 밀리고요.
```

→ 첫 단락에 "K뷰티 OEM" 키워드 + 문제 환기 + 검색 의도 (이 시간 줄이는 방법) 미리 시사

## N2. 케이스 스터디형 (1,500~2,000자, H2 4개)

**제목 패턴**: "{고객·사례} — {달성 결과}"

**구조**:
```
[제목]
{사례 + 결과 수치를 제목에 노출}

[도입 — 약 200자]
{어떤 회사·어떤 상황 + 결과 한 줄 스포일러}

[H2-1: 시작 시점의 상황 — 약 400자]
{Before — 무엇이 문제였는지}

[H2-2: 어떤 변화를 시도했는가 — 약 400자]
{무엇을 도입·바꿨는지}

[H2-3: 결과·수치 — 약 400자]
{After — 구체 수치, 변화 양상}

[H2-4: 적용 가이드·CTA — 약 200자]
{읽는 사람이 어떻게 동일 가치 얻을 수 있는지 + 링크}

[태그 10~15개]
```

## N3. 비교·리뷰형 (2,000~2,500자, H2 5~6개)

**제목 패턴**: "{대상 A} vs {대상 B} — {기준 키워드}"

**구조**:
```
[도입 — 약 200자]
{비교 배경 + 누가 이 결정을 고민하는가}

[H2-1: 비교 기준 — 약 300자]
{어떤 항목으로 비교하는가}

[H2-2: 대상 A 상세 — 약 500자]

[H2-3: 대상 B 상세 — 약 500자]

[H2-4: 항목별 비교 표 — 약 300자]
{한 화면 요약}

[H2-5: 어떤 경우에 어떤 선택 — 약 300자]
{추천 가이드}

[H2-6: 결론·CTA — 약 200자]

[태그 10~15개]
```

## N4. 인사이트·관점형 (1,500~2,000자, H2 3~4개)

**제목 패턴**: "{단정·반전 메시지} — {근거 키워드}"

**구조**:
```
[도입 — 약 250자]
{관점 단정 + 왜 그렇게 보는지 미리}

[H2-1: 통념 vs 우리의 관점 — 약 500자]
{대비 구조로 차별 메시지}

[H2-2: 근거·데이터·경험 — 약 600자]
{관점을 뒷받침하는 사실들}

[H2-3: 그래서 무엇을 의미하는가 — 약 400자]
{독자가 가져갈 implication}

[H2-4: 마무리·관련 콘텐츠 — 약 200자]

[태그 10~15개]
```

## N5. 제품 소개형 (1,500~2,000자, H2 4~5개)

**제목 패턴**: "{제품·서비스명} {기능 핵심} — {차별점·성과}"

**구조**:
```

{문제 환기 + 제품 한 줄 소개}

[H2-1: 어떤 문제를 푸는가 — 약 400자]
{P 단계 — 페르소나의 일상 문제}

[H2-2: 제품 핵심 — 약 400자]
{S 단계 — 어떻게 푸는지}

[H2-3: 핵심 기능 상세 — 약 500자]
{O 단계 — 구체 수치·기능}

[H2-4: 어떤 단계에서 유용한가 — 약 400자]
{사용 시나리오}

[H2-5: 시작하는 방법 — 약 200자]
{N+A — 대상 좁히기 + soft CTA}

[태그 10~15개]
```

## 채우기 규칙 (copywriter용)

1. `{...}` 자리는 회사 프로필의 톤으로 재작성. 템플릿 문장을 그대로 두지 말 것.
2. **제목**: 핵심 키워드를 앞 25자 안에 배치. 60자 이내. 광고문구형 어휘 X.
3. **첫 단락**: 핵심 키워드 1회 + 검색 의도 미리 시사.
4. **본문 키워드 분포**: 4~6회 자연스럽게 (stuffing 금지).
5. **H2 4~6개**: 각 H2 첫 줄에 결론·요약 (스캔 읽기 최적화).
6. **외부 링크**: 본문 첫 단락 X. 마지막 또는 자연스러운 위치만.
7. **태그 10~15개**: long-tail 키워드 위주 (단순 1단어 X).
8. **회사 톤은 유지하되 정보형으로 약간 객관화**: SNS보다 설명적.

## image-director 호출 안 함

네이버 블로그는 카드뉴스·이미지가 핵심이 아니므로 `image-director` 호출 skip. 본문 안에 이미지가 필요하면 사용자가 직접 첨부 (또는 후속 작업).
