# Naver Blog Strategy

네이버 블로그(블로그.네이버.com) 채널 마케팅 전략. 이 문서는 카피라이팅 단계(`copywriter` 서브에이전트)의 필수 입력이다.

> 이 채널은 **SNS가 아닌 검색 매체**. Threads/Instagram/LinkedIn과 본질이 다름. SEO·검색 의도 충족이 최우선.

## 한 줄 요약
**검색 의도 충족 + 키워드 자연 분포 + 1500자 이상 정보 밀도**. 광고 카피 X, 정보성 글 O. 한국 사용자가 네이버에서 검색했을 때 찾고 싶어 하는 답을 제공.

## 채널 특성

| 항목 | 내용 |
|------|------|
| 매체 유형 | 검색 기반 블로그 (SNS 아님) |
| 글자 수 | **1,500~3,000자 권장** (SEO 최적 구간), 한도 거의 없음 |
| 미디어 | 이미지 0~10장 옵션 (없어도 됨), 동영상 첨부 가능 |
| 구조 | H2/H3 사용 권장 (4~6개 H2) |
| 제목 | **45~60자**, 핵심 키워드 앞쪽 배치 |
| 태그 | 30개 한도, 10~15개 권장 (long-tail 키워드 위주) |
| 발행 시간 | 평일 10~12시, 14~16시 (검색 트래픽 피크) |
| API | XML-RPC API (Wordpress 호환) — 토큰 발급 후 사용 |

## SEO 핵심 원칙

1. **제목에 핵심 키워드 1~2개 앞쪽 배치**
   - "한국 화장품 OEM, 출장 없이 3초에 매칭하는 방법" (O — "한국 화장품 OEM" 앞)
   - "출장 없이 매칭하는 신박한 방법!" (X — 키워드 모호)
2. **첫 단락에 핵심 메시지 + 키워드**
   - 검색 결과 미리보기에 노출되는 부분
3. **본문 키워드 자연 분포 4~6회**
   - 키워드 stuffing 금지 (네이버 SEO 패널티)
4. **H2 4~6개 + H3 보조**
   - 구조화된 글이 노출 우선
5. **글자수 1,500자 이상**
   - 정보 밀도 부족 글은 노출 하락
6. **이미지·내부링크는 보조**
   - 텍스트 SEO가 우선, 이미지 alt text 권장

## 콘텐츠 원칙 (네이버 블로그 한정)

1. **검색 의도 충족이 최우선**
   - 사용자가 "한국 화장품 OEM 매칭"이라고 검색했을 때 찾는 답을 제공
2. **광고 외침 금지, 정보 우선**
   - "최고의" "1등" "유일한" 류 표현은 SEO·신뢰 모두 깎음
3. **두괄식 + 각 섹션 첫 줄에 결론**
   - 스캔 읽기 최적화
4. **외부 링크는 본문 마지막 또는 자연스러운 위치**
   - 첫 단락 외부 링크 X
5. **회사 프로필의 톤 유지하되, 정보형 변환**
   - SNS 톤보다 약간 더 객관적·설명적

## 금기 (이 채널 한정)

- 키워드 stuffing (같은 키워드 8회+ 반복)
- 광고문구형 도입 ("지금 신청하세요!!!", "한정 특가")
- 외부 링크를 본문 첫 단락에 배치 (SEO 페널티)
- 네이버 정책 위반 (성인·도박·불법 약물·과대 광고 등)
- 회사 프로필의 `banned.words / topics / claims` 위반
- 광고 분류 시 본문 시작에 "본 포스팅은 광고임을 알립니다" 또는 `legal.adHashtag` 표시

## 콘텐츠 타입별 권장 형식

| 타입 | 길이 | 구조 | 이미지 |
|------|------|------|--------|
| 가이드 | 2,000~2,500자 | H2 5~6개 (단계별) | 0~3장 (선택) |
| 케이스 스터디 | 1,500~2,000자 | H2 4개 (배경/문제/해결/결과) | 0~2장 |
| 비교·리뷰 | 2,000~2,500자 | H2 5~6개 (항목별) | 0~5장 |
| 인사이트·관점 | 1,500~2,000자 | H2 3~4개 (논리 단계) | 0~1장 |
| 제품 소개 | 1,500~2,000자 | H2 4~5개 (문제/제품/기능/사례/CTA) | 0~3장 |

## 회사 프로필 매핑

`copywriter`가 카피 생성 시:
- `tone.preset` + `tone.voiceNotes` → 본문 호흡 (단, 정보형으로 약간 객관화)
- `tone.sampleSentences` → 모방 학습용
- `targetAudience[].painPoints` → 도입부 문제 환기 + H2 항목 도출
- `keyMessage` → 첫 단락에 자연 삽입
- `contentPoints` → H2 본론 구성 재료
- `banned.*` → 사후 검열
- `hashtags.always` + `keywords` → 본문 자연 분포 + 태그 (10~15개)
- `legal.adDisclosureRequired` → 본문 시작 또는 끝에 광고 표시

**핵심**: SNS와 달리 카드뉴스·이미지가 핵심이 아님 → `image-director` 호출 **skip**. cardVisual 단계 없음.

## 업로드 (Phase 4 — Naver Blog API)

```
1. POST blog.naver.com/api/blog/post (XML-RPC 또는 OAuth)
   - title, content, tags, category
2. 응답: post_id, post_url
3. (선택) 분류 카테고리 지정
```

XML-RPC 인증: blogId + appId + appSecret. 자세한 어댑터 구현은 `harness/src/publisher/adapters/naver-blog.mjs` 참조.
