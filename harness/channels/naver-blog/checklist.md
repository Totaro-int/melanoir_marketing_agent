# Naver Blog 발행 전 체크리스트

휴먼 승인 게이트(`/sns-preview <slug>` → approve)에서 사용. `copywriter` 산출물에 대해 자동 + 수동 점검.

> 네이버 블로그는 SNS와 달리 **이미지 없이도 발행 가능**. `image-director` 점검은 skip.

## 자동 점검 (Phase 3 게이트가 차단)

### 본문
- [ ] 본문 길이 ≥ 1,500자 (네이버 SEO 최적 구간 하한)
- [ ] 본문 길이 ≤ 5,000자 (너무 길면 이탈률 상승)
- [ ] H2 헤딩 4~6개 (구조화)
- [ ] 핵심 키워드(첫 단락 + 본문)에 자연 분포 4~6회
- [ ] **첫 단락(150자 내)**에 핵심 키워드 1개 이상 포함
- [ ] `banned.words` 미포함
- [ ] `banned.claims` 미포함 ("최고의", "1위", "유일한", "100%" 등)
- [ ] `banned.topics` 미포함

### 제목
- [ ] 제목 길이 ≤ 60자 (네이버 검색 결과 노출 최적)
- [ ] 제목 길이 ≥ 25자 (너무 짧으면 SEO 약함)
- [ ] 제목에 **핵심 키워드 1개 이상 앞부분(앞 25자 이내)** 배치
- [ ] 제목에 광고문구형 어휘 없음 (느낌표 X, "지금" 단독 X)

### 태그·메타
- [ ] 태그 10~15개
- [ ] 태그 long-tail 키워드 위주 (단순 1단어 키워드 X)
- [ ] (광고 분류 시) 본문 시작 또는 끝에 광고 표시

## 수동 점검 (사람이 확인)

### SEO·검색 의도
- [ ] 사용자가 무엇을 검색했을 때 이 글이 답이 되는지 명확한가?
- [ ] 제목·첫 단락이 그 검색 의도와 일치하는가?
- [ ] 키워드 stuffing처럼 어색한 부분 없는가? (자연스러운 분포)

### 본문
- [ ] 회사 톤(`tone.voiceNotes`)과 일치하는가? (정보형으로 약간 객관화 OK)
- [ ] 광고 외침이 아닌 정보 전달인가?
- [ ] 사실 확인 — 숫자·인용·날짜 정확성
- [ ] H2 각 섹션 첫 줄에 결론·요약이 있는가? (스캔 읽기 최적화)
- [ ] CTA가 본문 끝에 자연스럽게 들어갔는가? (강요 X)

### 추가
- [ ] 외부 링크가 본문 첫 단락에 없는가? (SEO 페널티)
- [ ] 이미지 첨부 시 alt text 적절한가? (접근성·SEO)

## 실패 시 행동

- 자동 점검 실패: 사용자에게 어느 룰이 깨졌는지 표시 + 자동 수정 제안
- 수동 점검에서 사용자가 reject: 사용자가 적은 사유를 `copywriter`에 피드백으로 전달해 재생성
