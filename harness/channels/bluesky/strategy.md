# Bluesky Strategy

Decentralized 마이크로블로깅 (AT Protocol). 분위기는 초기 Twitter + 개발자/디자이너 비중 높음.

## 핵심 제약

| 항목 | 한도 |
|------|------|
| 본문 | 300자 (그래픽 본문 내 카운트는 byte 기반) |
| 이미지 | 최대 4장 (각 1MB 이하 권장) |
| 영상 | (지원 안 함 — 별도 호스팅 링크) |
| 해시태그 | inline 1~2개가 자연스러움 (Twitter 처럼 spam 톤 금지) |

## 톤 가이드

- **마케팅 카피 톤 금지** — Bluesky 사용자 거부감 큼
- **개발자 노트** 또는 **현장 코멘트** 톤이 받음
- 짧고 직접적. 본문 자체가 결론, "더 보기"·"링크 클릭" 유도 X
- 이모지 1~2개 OK, 과하면 X
- 링크 미리보기 (embed.external) 자동 unfurl — 본문에서 URL 한 번만 언급

## 어울리는 콘텐츠

- 제품 업데이트 한 줄
- 작은 인사이트 / 데이터 포인트
- 팀의 일상 컷 (사진 1장 + 캡션)

## 어울리지 않는 콘텐츠

- 길게 풀어쓴 케이스 스터디 (LinkedIn/Threads 로)
- 카드뉴스 식 4컷 narrative (Threads 캐러셀로)
- "지금 무료 체험!" 류 직접 광고
