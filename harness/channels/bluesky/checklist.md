# Bluesky 발행 전 체크리스트

휴먼 승인 게이트(`/sns-preview <slug>` → approve)에서 사용. `copywriter` + `image-director` 산출물에 대해 자동 + 수동 점검.

## 자동 점검 (Phase 3 게이트가 차단)

### 텍스트
- [ ] 본문 300자 이하 (한글 byte 카운트 주의)
- [ ] 한국어는 ~150자 권장 (단정·관점)
- [ ] 첫 줄이 명확한 단정·관점인가?
- [ ] 줄바꿈 1~2개 이하 (Bluesky 톤 — 한 호흡)
- [ ] 해시태그 0~2개 (도배 시 외면)
- [ ] `banned.words / claims / topics` 미포함
- [ ] (선택) `hashtags.always` 1개만 또는 생략

### 이미지
- [ ] 0~4장 범위 (1~2장 권장)
- [ ] 각 < 1MB
- [ ] 1:1 또는 16:9 또는 4:5 자유
- [ ] 캐주얼·소셜 톤 (카드뉴스 임팩트 X)
- [ ] (광고 분류 시) `legal.adHashtag` 포함

### 봇 패턴 회피
- [ ] 정확한 시간 발행 X (분 단위 변동 OK)
- [ ] 정형 형식 반복 X (이모지 도배·정형 머리말)
- [ ] AI 티 나는 어휘 X ("delve", "혁신적", "최고의")

## 수동 점검

### 톤
- [ ] 캐주얼·진솔한가? (광고 톤 0건)
- [ ] 사람 목소리인가? 봇처럼 들리지 않는가?
- [ ] 1인칭 자연스러운가?
- [ ] 회사 톤(`tone.voiceNotes`)을 한 단계 친근하게 변환했는가?

### 상호작용
- [ ] 답글이 달릴 만한 여지가 있는가?
- [ ] 짧은 농담·관점이 받을 만한가?
- [ ] 링크 unfurl 미리보기가 매력적인가?

## 실패 시 행동

- 자동 실패: 어느 룰 깨졌는지 표시 + 자동 수정 제안
- 수동 reject: 사유를 copywriter에 피드백 → 재생성 (Bluesky 캐주얼 톤 강조)
