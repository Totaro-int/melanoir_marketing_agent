# copy-deck — 복사 붙여넣기 덱 (봇 의심 0 발행)

브라우저 자동 발행(`browser-publish`)이 플랫폼에서 **봇으로 의심**받을 때 쓰는 대안.
자동화 없음 — 채널별 완성 카피를 띄워주고, **사람이 [복사]→플랫폼에 붙여넣기→직접 게시**한다.

## 두 가지 진입 (같은 화면)

**① 모니터링 대시보드 탭 (권장 — start-demo 쓰면 이미 떠 있음)**
대시보드(`localhost:7777`) 왼쪽 사이드바 **📝 복붙 덱** 탭. 오늘 캠페인의 채널별 글쓰기 칸이 바로 뜬다.
별도 명령 불필요 — `npm run morning` / `start-demo` 로 대시보드가 떠 있으면 탭만 누르면 됨.

**② 단독 서버 (대시보드 안 띄울 때)**
```bash
npm run deck            # 가장 최근 draft 있는 캠페인
npm run deck <slug>     # 특정 캠페인
# → http://localhost:7788 (포트 변경: -- --port=7790)
```

## 페이지에 뭐가 있나 (채널별 카드)
- **글쓰기 칸**(textarea) — 완성된 본문(+해시태그). `[📋 본문 복사]` 한 번이면 클립보드로.
- `[#태그만 복사]` — 해시태그만 따로.
- `[플랫폼 열기 ↗]` — Instagram/Threads/LinkedIn/네이버블로그 새 탭.
- **첨부 이미지** 썸네일 — 클릭하면 저장/열기(드래그로 첨부). 블로그는 섹션 순서대로.
- 채널 순서: 블로그(본문) 먼저 → 소셜.

## 쓰는 흐름 (사람)
1. `npm run deck` → 브라우저에서 열기
2. 채널 카드에서 `[본문 복사]`
3. `[플랫폼 열기]` → 새 글 작성 → **붙여넣기**
4. 이미지 썸네일 저장 → 첨부
5. 직접 [게시] (← 사람이 누름. 자동화 아님)

## browser-publish 와 비교
| | browser-publish | copy-deck |
|---|---|---|
| 발행 | 브라우저 자동화(크롬 9222) | 사람이 복붙·게시 |
| 봇 의심 | 있을 수 있음 | **없음** |
| 속도 | 빠름(반자동) | 수동(채널당 30초~1분) |
| 용도 | 신뢰된 계정·대량 | **의심받는 계정·신규·민감 채널** |

둘 다 남겨둔다 — 계정/채널 상황에 맞게 선택. 생성 파이프라인(generate/finalize)은 공통.

## 안전
- 로컬 전용(127.0.0.1). 이미지는 그 캠페인 `draft.assets` 에 등록된 파일만 서빙(경로 검증 — 임의 파일 접근 차단).
- 발행 계정 자격증명 일절 안 다룸. 순수 복붙 보조.
